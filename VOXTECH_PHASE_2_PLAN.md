# VOXTECH_PHASE_2_PLAN.md

## Project
VoxTech Authority and Proof Expansion

## Phase Objective
Expand the initial marketing website into a stronger authority and proof platform by deepening portfolio storytelling, improving content discoverability, and introducing conversion trust assets.

## Strategic Goal
Phase 2 is about strengthening trust and increasing conversion confidence.

If Phase 1 establishes presence, Phase 2 must establish depth.

Visitors should not only believe that VoxTech exists.
They should believe VoxTech understands business problems, builds real solutions, and is worth speaking to.

## Primary Business Outcomes
- increase demo conversion readiness
- improve organic discoverability
- strengthen decision-maker confidence
- extend average session depth
- turn the website into a stronger authority engine

## Phase 2 Scope
This phase should add:
- detailed case study system
- richer work storytelling
- blog filtering and content architecture upgrades
- opportunities filtering and timeline controls
- testimonials and social proof
- downloadable capability or company profile asset
- stronger lead magnets
- stronger internal linking and topical authority

## Phase 2 Deliverables
1. Case study framework
2. Testimonials and trust section system
3. Search and filter improvements
4. Content hubs and category architecture
5. Downloadable capability deck or company profile
6. Authority-focused homepage and page refinements
7. Better lead nurturing pathways

## Expanded User Journey
Home -> What We Do -> Work -> Case Study -> Book a Demo
Insights -> category exploration -> related projects -> Book a Demo
Capability Deck -> trust reinforcement -> Contact or Book a Demo

## Functional Requirements

### 1. Case Studies
Move beyond project cards and create full story-driven pages.

#### Case Study Requirements
Each case study page should include:
- project overview
- client or sector context
- challenge
- strategy
- solution
- implementation highlights
- business outcome
- relevant screenshots
- key technologies
- CTA to discuss a similar build

#### Case Study Use Cases
- ClinicOS case study
- tseboIQ case study
- ScanYa case study

### 2. Testimonials and Social Proof
Introduce proof blocks across the site.

#### Requirements
- testimonial schema
- name
- role
- company
- testimonial text
- optional company logo
- optional featured flag

#### UI Use Cases
- homepage trust strip
- service page quote blocks
- work page trust modules
- final CTA reinforcement

### 3. Insights Upgrade
Improve discoverability and depth.

#### Requirements
- category pages
- article filtering
- featured articles
- author pages or author snippets
- related content engine
- reading time support
- share actions
- SEO refinement

#### Goal
Make Insights feel like a true editorial and knowledge system.

### 4. Opportunities Upgrade
Introduce better event and collaboration browsing.

#### Requirements
- filter by type
- filter by location
- upcoming and past views
- featured opportunity support
- external registration handling
- collaboration call format
- event detail improvements

### 5. Downloadable Capability Asset
Create a polished downloadable capability deck or company profile.

#### Asset Requirements
- who VoxTech is
- what VoxTech does
- featured services
- selected work
- why work with VoxTech
- contact and booking details

#### Purpose
This supports sales conversations and makes the brand easier to share.

### 6. Lead Magnets and Conversion Enhancements
Introduce value-led conversion assets.

#### Potential Lead Magnets
- SME software readiness checklist
- MVP planning guide
- internal workflow modernization checklist
- product discovery worksheet

#### Functional Requirements
- landing page support
- gated or ungated download logic
- capture form support if needed
- CTA linking from blog and services

## Acceptance Criteria
Phase 2 is complete when:
- case study pages exist
- testimonials can be managed via CMS
- insights and opportunities support filters or categorization enhancements
- downloadable capability content is available
- trust and proof are stronger across the site
- users have more reasons to move toward demo booking

## Risks
- creating content depth without enough actual content
- overcomplicating navigation
- adding too many secondary CTAs
- making the site feel editorial but less commercial

## Execution Priority
